# Политика безопасности

## Поддерживаемые версии

Мы предоставляем обновления безопасности для следующих версий:

| Версия | Поддержка          |
| ------ | ------------------ |
| 2.3.x  | :white_check_mark: |
| 2.2.x  | :white_check_mark: |
| 2.1.x  | :x:                |
| < 2.0  | :x:                |

## Сообщение об уязвимости

Если вы обнаружили уязвимость в безопасности, пожалуйста, **не создавайте публичный issue**.

Вместо этого:

1. Создайте приватный issue на Codeberg с меткой "security"
2. Опишите уязвимость максимально подробно:
   - Тип уязвимости
   - Шаги для воспроизведения
   - Потенциальное влияние
   - Предлагаемое решение (если есть)

## Что мы считаем уязвимостью

- Выполнение произвольного кода
- Обход прав доступа
- SQL инъекции (если применимо)
- Утечка конфиденциальных данных
- Отказ в обслуживании (DoS)
- Дюпы предметов или ресурсов

## Что НЕ является уязвимостью

- Баги, не связанные с безопасностью
- Проблемы производительности
- Несовместимость с другими плагинами
- Запросы на новые функции

## Процесс обработки

1. **Подтверждение** - мы подтвердим получение вашего сообщения в течение 48 часов
2. **Оценка** - мы оценим серьезность и влияние уязвимости
3. **Исправление** - мы разработаем и протестируем исправление
4. **Релиз** - мы выпустим обновление безопасности
5. **Раскрытие** - после выпуска исправления мы опубликуем информацию об уязвимости

## Благодарности

Мы благодарим всех, кто ответственно сообщает об уязвимостях и помогает сделать LoUtils более безопасным.
