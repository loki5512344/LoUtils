# LoUtils

Многофункциональный плагин для Paper/Folia 1.21.3+

**Автор:** loki  
**Версия:** 2.2.0

## 📦 Установка

1. Установите **Java 21** (обязательно!)
   - Скачать: https://adoptium.net/temurin/releases/?version=21
2. Скачайте `LoUtils-2.2.0.jar`
3. Поместите в папку `plugins/`
4. Перезапустите сервер

## 🔧 Зависимости

- **Paper/Folia 1.21.3+** (обязательно)
- **Java 21** (обязательно!)
- **PlaceholderAPI** (опционально)
- **LuckPerms** (опционально)

### 🍃 FastLeafDecay - Быстрое гниение листвы

Листва исчезает за 2 секунды после рубки дерева.

**Особенности:**
- Удаляет только листву срубленного дерева (не трогает соседние деревья)
- Определяет тип дерева (дуб, берёза, ель и т.д.)
- Правильный дроп саженцев и яблок
- Красивая анимация исчезновения
- Умный алгоритм: если часть дерева осталась - листва на ней не исчезнет

**Команды:**
- `/lfastleaf enable` - включить
- `/lfastleaf disable` - выключить
- `/lfastleaf status` - статус
- `/lfastleaf reload` - перезагрузить конфиг
- `/lfastleaf set <delay|radius|animation>` - настроить параметры

**Конфиг:** `conf/fastleafdecay.yml`
```yaml
enabled: true
decay-delay: 40        # Задержка перед гниением (тики)
search-radius: 5       # Радиус поиска листвы
animation-delay: 2     # Задержка между удалением листьев
smart-decay: true      # Умный алгоритм
```

### 🔥 CauldronCrafting - Котел как станция переработки

Котел становится станцией для переработки предметов.

**Способы использования:**
- **ПКМ по котлу** с предметом в руке - предмет заменяется в руке
- **Бросить предмет** в котел (Q) - новый предмет выпадает рядом

**Рецепты:**

1. **Очистка бетона**
   - Цементный порошок → Бетон
   - Работает со всеми 16 цветами
   - Эффекты: пузыри + звук шипения

2. **Стирка (удаление краски)**
   - Цветной предмет → Белый/дефолтный цвет
   - Работает с:
     - Кожаная броня (удаляет краску)
     - Флаги (любой цвет → белый флаг)
     - Кровати (любой цвет → белая кровать)

**Особенности:**
- Расходует 1 уровень воды из котла
- Работает только с котлами, наполненными водой
- Мгновенная переработка
- Два способа использования: ПКМ или бросок

**Конфиг:** `conf/cauldron.yml`
```yaml
concrete-cleaning:
  enabled: true
  water-cost: 1
washing:
  enabled: true
  water-cost: 1
```

### 💡 LightBlock - Блок света

Невидимый источник света с настраиваемым уровнем освещения.

**Крафт:**
```
[Светокамень] [Светокамень] [Светокамень]
[Светокамень] [  Свечка  ] [Светокамень]
[Светокамень] [Светокамень] [Светокамень]
= Блок света (уровень 15)
```

**Использование:**
- **ПКМ** - поставить блок света
- **Shift + ПКМ по блоку** - изменить уровень света (0-15)
- **ПКМ по блоку с блоком света в руке** - удалить блок и получить дроп
- **Shift + ПКМ в воздух** - изменить уровень блока в руке
- Невидимый, но светит
- Сохраняет уровень света при удалении

**Видимость блоков света:**
- **Креатив/Спектатор** - видят блоки нативно (как барьеры)
- **Выживание/Приключение** - видят частицы GLOW вокруг блоков когда держат блок света в руке
- Работает как барьеры: держишь в руке = видишь все блоки света в радиусе 16 блоков
- Частицы показывают 8 углов каждого блока света

**Как удалить блок света в выживании:**
1. Возьми блок света в руку
2. Увидишь частицы вокруг всех блоков света
3. Кликни ПКМ по блоку света который хочешь удалить
4. Блок удалится и выпадет как предмет

**Особенности:**
- Идеально для декоративного освещения
- Простое удаление (ПКМ с блоком в руке)
- Можно настроить любой уровень света (0-15)
- Эффективная система частиц (только когда держишь в руке)

**Конфиг:** `conf/light-block.yml`
```yaml
show-particles: true    # Показывать частицы когда держишь в руке
particle-radius: 16     # Радиус видимости частиц
```

### 🖼️ InvisibleFrames - Невидимые рамки

Рамки становятся полностью невидимыми (остаётся только предмет).

**Крафты:**

1. **Верстак:**
```
[Рамка] [Рамка] [Рамка]
[Рамка] [Зелье ] [Рамка]
[Рамка] [Рамка] [Рамка]

Зелье невидимости + 8 рамок = 8 невидимых рамок
```
Работает с любым зельем невидимости:
- Обычное зелье (Potion of Invisibility)
- Splash зелье (Splash Potion of Invisibility)
- Lingering зелье (Lingering Potion of Invisibility)

2. **Splash/Lingering зелье:**
   - Бросьте splash или lingering зелье невидимости на рамку
   - Рамка станет невидимой

**Особенности:**
- Работает с обычными и светящимися рамками
- Предмет в рамке остаётся видимым
- Идеально для карт и декора
- Название в инвентаре: "Невидимая рамка" (настраивается)

**Конфиг:** `conf/invisible-frames.yml`
```yaml
display-name: "&#B798A8Невидимая рамка"  # Название предмета
splash-potion-method: true                # Метод с броском зелья
crafting-method: true                     # Метод с крафтом
```

### 🎭 Player Poses - Система поз

Позволяет игрокам сидеть, лежать и ползать (аналог GSit).

**Команды:**
- `/sit` - сесть на месте
- `/pose stop` - встать из позы

**Автосидение:**
- ПКМ по ступенькам/плитам/коврам = автоматически сесть
- ПКМ по игроку = сесть на его голову
- Настраиваемый список блоков в конфиге

**Особенности:**
- Сидение использует невидимый ArmorStand
- Автоотмена при движении или получении урона
- Все сообщения в action bar (не спамят чат)
- Правильная телепортация при вставании (+1 блок вверх)

**Права:**
- `loutils.sit` - доступ к /sit (по умолчанию всем)
- `loutils.pose` - доступ к /pose (по умолчанию всем)

**Конфиг:** `conf/poses.yml`
```yaml
sit:
  enabled: true
  right-click-enabled: true
  sit-height: -1.5
general:
  cancel-on-damage: true
  cancel-on-move: true
player-sit-enabled: true  # Сидение на голове игрока
```

### 🗺️ Map Locking - Блокировка карт

Защита карт от копирования с сохранением информации о владельце.

**Команды:**
- `/map lock` - заблокировать карту (только владелец может разблокировать)
- `/map unlock` - разблокировать карту (только владелец)

**Механика:**
- При блокировке в NBT карты сохраняется UUID владельца
- В описании карты добавляется строка с именем владельца
- Попытка разблокировать чужую карту показывает сообщение
- Заблокированную карту нельзя скопировать в картографическом столе
- Защита мапартов от нежелательного копирования

**Особенности:**
- Все сообщения в action bar
- Блокировка через NBT теги (персистентность)
- Идеально для защиты мапартов

**Конфиг:** `conf/map-locking.yml`

### 🖼️ Frame Locking - Блокировка рамок

Защита рамок от поворота и разрушения с помощью мёда.

**Механика:**
- ПКМ по рамке с мёдом = заблокировать рамку
- ПКМ по заблокированной рамке с кистью = разблокировать
- Заблокированную рамку нельзя:
  - Повернуть предмет внутри
  - Достать предмет
  - Сломать саму рамку

**Эффекты:**
- Звук: ITEM_HONEYCOMB_WAX_ON при блокировке
- Звук: ITEM_AXE_WAX_OFF при разблокировке
- Частицы мёда при блокировке
- Частицы воска при разблокировке

**Особенности:**
- Работает с обычными и светящимися рамками
- Защита от гриферов
- Все сообщения в action bar

**Конфиг:** `conf/frame-locking.yml`

### 🔨 Anvil Repair - Починка наковален

Починка повреждённых наковален железными блоками.

**Механика:**

**Эффекты:**
- Звук: BLOCK_ANVIL_USE
- Частицы железа (ITEM с железным блоком)

**Особенности:**
- Экономия ресурсов

- Все сообщения в action bar

**Конфиг:** `conf/anvil-repair.yml`

### 🔄 AutoRestart - Авто-рестарт

Автоматический рестарт сервера по таймеру.

**Команды:**
- `/lar start` - запустить таймер
- `/lar stop` - остановить
- `/lar status` - статус
- `/lar reload` - перезагрузить

**Особенности:**
- Предупреждения за 30, 15, 10, 5, 3, 2, 1 минуту
- Предупреждения за 30, 15, 10, 5 секунд
- Сохранение миров перед рестартом
- Folia compatible

**Конфиг:** `conf/autorestart.yml`

---

### 💀 Death Messages - Сообщения смерти

Кастомные сообщения о смерти игроков.

**Особенности:**
- Настраиваемые сообщения для каждого типа смерти
- Поддержка плейсхолдеров
- Цветовое форматирование

**Конфиг:** `conf/deathmessages.yml`

---

### 📈 PerformanceProfiler - Мониторинг производительности

Автоматический мониторинг TPS с отправкой отчётов в Discord.

**Особенности:**
- Проверка TPS каждые 30 секунд
- Отправка отчёта при падении TPS ниже порога
- Детальная информация:
  - Текущий TPS и MSPT
  - Список энтити по типам (топ 20)
  - Список игроков с координатами (топ 10)
  - Анализ чанков с большим количеством энтити
- Cooldown между отчётами (300 сек)

**Настройка:**
1. Создайте Discord вебхук
2. Вставьте URL в `conf/performance.yml`
3. Настройте порог TPS
4. Перезапустите или `/loutils reload`

**Конфиг:** `conf/performance.yml`

## 📐 Архитектура

Проект следует принципам **SOLID/DRY/KISS**:

- **ConfigLoader** - загрузка конфигов
- **MessageService** - работа с сообщениями
- **CommandBase** - базовый класс команд
- **BaseStorageManager** - базовый класс хранилищ
- **DependencyContainer** - dependency injection

**Структура:**
```
src/main/java/xyz/lokili/loutils/
├── api/              # Интерфейсы
├── commands/         # Команды
├── constants/        # Константы
├── core/             # Ядро (DI контейнер)
├── factories/        # Фабрики
├── listeners/        # Event listeners
├── managers/         # Менеджеры
├── placeholders/     # PlaceholderAPI
├── registry/         # Регистрация команд/листенеров
├── services/         # Сервисы
└── utils/            # Утилиты
```
